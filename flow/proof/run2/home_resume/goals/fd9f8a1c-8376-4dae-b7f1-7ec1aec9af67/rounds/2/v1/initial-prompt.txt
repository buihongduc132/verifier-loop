## Verify Claim Methodology (Evidence Chain Pattern)

When SAI/subagent makes a CLAIM about bugs/errors/problems, ALWAYS verify with evidence chain:

### The Questions to Ask

1. **"Where is the documentation?"** - Find official source
2. **"Is it official?"** - Verify authority/source
3. **"Known, where, whom remotely state that bug?"** - Find third-party confirmation

### Required Evidence Chain

```
Claim: "library has bug X"
        â
Required: Documentation OR
          Official issue tracker OR  
          Multiple user reports with links
        â
Verification: Find exact source citing bug
```

### Example Pattern

| Claim | What to Find |
|-------|--------------|
| `AlreadyCalledError` bug in library | GitHub issues / Forum posts / Stack Overflow |
| "API is down" | Official status page / Twitter/X announcement |
| "Security vulnerability" | CVE / Security advisory / Official issue |

### How to Verify

1. **Search GitHub issues** - repo/issues
2. **Search community forums** - official support forums
3. **Search Stack Overflow** - developer discussions
4. **Check official docs** - changelog / known issues

### Anti-Pattern to Catch

- SAI cites a "bug" but provides no links
- "I saw it somewhere" without evidence
- General "library has issues" without specifics

### Tone/Style

Ask with investigator mindset:
- "Where did you find this information?"
- "Can you link me to the issue/report?"
- "Who else has reported this?"
